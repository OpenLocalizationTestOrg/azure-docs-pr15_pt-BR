# <a name="publish-azure-media-services-content-using-rest"></a>Publicar conteúdo de serviços de mídia do Azure usando REST

> [AZURE.SELECTOR]
- [.NET](media-services-deliver-streaming-content.md)

##<a name="overview"></a>Visão geral


Você pode transmitir uma taxa de bits adaptativa MP4 definir criando um localizador de streaming OnDemand e criação de uma URL de streaming. O tópico de [codificação de um ativo](media-services-rest-encode-asset.md) mostra como codificar em uma taxa de bits adaptativa que MP4 definir. Se seu conteúdo for criptografado, configure política de entrega de ativos (conforme descrito [neste](media-services-rest-configure-asset-delivery-policy.md) tópico) antes de criar um localizador. 

Você também pode usar um OnDemand streaming localizador para construir URLs que apontam para arquivos MP4 que podem ser baixados de cada vez.  

Este tópico mostra como criar um OnDemand streaming localizador para publicar seu ativo e criar uma suave, MPEG traço e HLS streaming URLs. Ele também mostra quente para construir URLs de download progressivo.

A seção a [seguir](#types) mostra os tipos de enumeração cujos valores são usados nas chamadas REST.   
  
##<a name="create-an-ondemand-streaming-locator"></a>Criar um OnDemand streaming localizador

Para criar o OnDemand streaming localizador e obter URLs, você precisa fazer o seguinte:


   1. Se o conteúdo é criptografado, defina uma política de acesso.
   2. Crie um OnDemand streaming localizador.
   3. Se você planeja transmitir, obtenha o arquivo de manifesto streaming (.ism) no ativo. 
        
    Se você planeja download progressivo, obter os nomes dos arquivos MP4 no ativo. 
   4. Construir URLs para o arquivo manifesto ou arquivos MP4. 
   5. Observe que não é possível criar um localizador de streaming usando um AccessPolicy que inclui escrever ou excluir as permissões.

###<a name="build-streaming-urls"></a>Construir streaming URLs

Use o valor de **caminho** retornado após a criação do localizador de criar o suave, HLS e URLs de traço MPEG. 

Suave Streaming: **Caminho** + o nome do arquivo manifesto + "/ manifesto"

exemplo:

    http://amstest1.streaming.mediaservices.windows.net/3c5fe676-199c-4620-9b03-ba014900f214/BigBuckBunny.ism/manifest

HLS: **Caminho** + manifesto nome do arquivo + "/ manifest(format=m3u8-aapl)"

exemplo:

    http://amstest1.streaming.mediaservices.windows.net/3c5fe676-199c-4620-9b03-ba014900f214/BigBuckBunny.ism/manifest(format=m3u8-aapl)


TRAÇO: **Caminho** + nome do arquivo manifesto + "/ manifest(format=mpd-time-csf)"


exemplo:

    http://amstest1.streaming.mediaservices.windows.net/3c5fe676-199c-4620-9b03-ba014900f214/BigBuckBunny.ism/manifest(format=mpd-time-csf)


###<a name="build-progressive-download-urls"></a>Construir URLs de download progressivo

Use o valor de **caminho** retornado após a criação do localizador de para construir a URL de download progressivo.   

URL: **Caminho** + ativo nome do arquivo mp4

exemplo:

    http://amstest1.streaming.mediaservices.windows.net/3c5fe676-199c-4620-9b03-ba014900f214/BigBuckBunny_H264_650kbps_AAC_und_ch2_96kbps.mp4
