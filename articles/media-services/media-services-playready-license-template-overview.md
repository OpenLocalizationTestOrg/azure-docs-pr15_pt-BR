#<a name="media-services-playready-license-template-overview"></a>Visão geral do modelo de licença PlayReady de serviços de mídia

Serviços de mídia do Azure agora fornece um serviço para fornecer o Microsoft PlayReady licenças. Quando o player de usuário final (por exemplo, Silverlight) tenta reproduzir seu conteúdo PlayReady protegido, uma solicitação é enviada para o serviço de entrega de licença para obter uma licença. Se o serviço de licença aprovar a solicitação, ele emite a licença que é enviada para o cliente e pode ser usada para descriptografar e reproduzir o conteúdo especificado.

Serviços de mídia também fornece APIs que permitem que você configure suas licenças PlayReady. Licenças contêm os direitos e restrições que você deseja para o runtime PlayReady DRM impor quando um usuário estiver tentando protegida de reprodução de conteúdo.
Abaixo estão alguns exemplos de restrições de licença PlayReady que você pode especificar:

- O DateTime do qual a licença é válida.
- O valor de DateTime quando a licença expira. 
- Para obter a licença seja salvo no armazenamento persistente no cliente. Licenças persistentes geralmente são usadas para permitir a reprodução offline do conteúdo.
- O nível de segurança mínima que deve ter um player para reproduzir seu conteúdo. 
- O nível de proteção de saída para os controles de saída para o conteúdo de audio\video. 
- Para obter mais informações, consulte a seção de controles de saída (3,5) no documento [PlayReady regras de conformidade](https://www.microsoft.com/playready/licensing/compliance/) .

>[AZURE.NOTE]Atualmente, você só pode configurar o PlayRight da licença PlayReady (esse direito é necessário). O PlayRight permite que o cliente para reproduzir o conteúdo. O PlayRight também permite configurar restrições específicas para reprodução. Para obter mais informações, consulte [PlayReadyPlayRight](media-services-playready-license-template-overview.md#PlayReadyPlayRight).

Para configurar licenças PlayReady usando os serviços de mídia, você deve configurar o modelo de licença PlayReady de serviços de mídia. O modelo é definido em XML.

O exemplo a seguir mostra o modelo mais simples (e mais comuns) que configura uma licença de streaming básica. Com esta licença, seus clientes poderão reprodução seu PlayReady conteúdo protegido.
    
    <?xml version="1.0" encoding="utf-8"?>
    <PlayReadyLicenseResponseTemplate xmlns:i="http://www.w3.org/2001/XMLSchema-instance" 
                                      xmlns="http://schemas.microsoft.com/Azure/MediaServices/KeyDelivery/PlayReadyTemplate/v1">
      <LicenseTemplates>
        <PlayReadyLicenseTemplate>
          <ContentKey i:type="ContentEncryptionKeyFromHeader" />
          <PlayRight />
        </PlayReadyLicenseTemplate>
      </LicenseTemplates>
    </PlayReadyLicenseResponseTemplate>

O XML está de acordo com o esquema XML do modelo da licença de PlayReady definido no modelo de licença do PlayReady seção de esquema XML.

Serviços de mídia também define um conjunto de classes .NET que pode ser usado para serializado e desserializado para e do XML. Para descrição das classes principais, consulte [classes .NET de serviços de mídia](media-services-playready-license-template-overview.md#classes). que são usados para configurar modelos de licença.

Para obter um exemplo de ponta a ponta que usa classes .NET para configurar o modelo de licença PlayReady, consulte [usando criptografia dinâmico do PlayReady e serviço de entrega de licença](https://msdn.microsoft.com/library/azure/dn783467.aspx).

##<a id="classes"></a>Classes .NET de serviços de mídia que são usadas para configurar modelos de licença

A seguir estão que as classes principais do .NET são usadas para configurar modelos de licença PlayReady de serviços de mídia. Essas classes mapeiam para os tipos definidos no [esquema XML de modelo de licença PlayReady](media-services-playready-license-template-overview.md#schema).

A classe [MediaServicesLicenseTemplateSerializer](https://msdn.microsoft.com/library/azure/microsoft.windowsazure.mediaservices.client.contentkeyauthorization.mediaserviceslicensetemplateserializer.aspx) é usada para serializar e desserializar de e para o modelo de licença de serviços de mídia XML.

###<a name="playreadylicenseresponsetemplate"></a>PlayReadyLicenseResponseTemplate

[PlayReadyLicenseResponseTemplate](https://msdn.microsoft.com/library/azure/microsoft.windowsazure.mediaservices.client.contentkeyauthorization.playreadylicenseresponsetemplate.aspx) - essa classe representa o modelo para a resposta enviada de volta para o usuário final. Ele contém um campo para uma cadeia de caracteres de dados personalizados entre o servidor de licenças e o aplicativo (pode ser útil para lógica de aplicativo personalizado), bem como uma lista de modelos de licença de um ou mais.

Esta é a classe de "nível superior" na hierarquia de modelo. Significando que o modelo de resposta inclui uma lista de modelos de licença e os modelos de licença incluem (direta ou indiretamente) todas as outras classes que compõem os dados de modelo para ser serializado.


###<a name="playreadylicensetemplate"></a>PlayReadyLicenseTemplate

[PlayReadyLicenseTemplate](https://msdn.microsoft.com/library/azure/microsoft.windowsazure.mediaservices.client.contentkeyauthorization.playreadylicensetemplate.aspx) - a classe representa um modelo de licença para a criação de licenças PlayReady a ser retornado para os usuários finais. Ele contém os dados sobre a chave de conteúdo da licença e qualquer direitos ou restrições impostas pelo runtime PlayReady DRM ao usar a chave de conteúdo.


###<a id="PlayReadyPlayRight"></a>PlayReadyPlayRight

[PlayReadyPlayRight](https://msdn.microsoft.com/library/azure/microsoft.windowsazure.mediaservices.client.contentkeyauthorization.playreadyplayright.aspx) - essa classe representa o PlayRight de uma licença de PlayReady. Ele concede ao usuário a capacidade de reprodução de conteúdo sujeito as restrições de zero ou mais configurado na licença e em PlayRight (para política específicas de reprodução). Muita a política na PlayRight tem fazer com as restrições de saída que controla os tipos de saídas que o conteúdo pode ser reproduzido sobre e alguma restrição que deve ser colocada no lugar ao usar um determinado de saída. Por exemplo, se o DigitalVideoOnlyContentRestriction estiver habilitada, então o tempo de execução DRM só permitirá o vídeo seja exibida sobre saídas digitais (analógicas saídas de vídeo não será possível passar o conteúdo).

>[AZURE.IMPORTANT]Esses tipos de restrições podem ser muito poderosos, mas também podem afetar a experiência do consumidor. Se a proteção de saída configuradas muito restritiva, o conteúdo pode ser podem fazê-lo em alguns clientes. Para obter mais informações, consulte o documento de [Regras de conformidade do PlayReady](https://www.microsoft.com/playready/licensing/compliance/) .

Para um exemplo de proteção níveis de suporte para o Silverlight, consulte: [Silverlight oferece suporte para proteção de saída](http://go.microsoft.com/fwlink/?LinkId=617318).
