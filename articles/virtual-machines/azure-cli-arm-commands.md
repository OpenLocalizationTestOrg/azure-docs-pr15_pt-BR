# <a name="azure-cli-commands-in-resource-manager-mode"></a>Azure comandos no modo do Gerenciador de recursos

Este artigo fornece opções para comandos do Azure interface de linha (comando) normalmente você usaria para criar e gerenciar recursos Azure no modelo de implantação do Gerenciador de recursos do Azure e sintaxe. Você pode acessar esses comandos executando a CLI no modo do Gerenciador de recursos (arm). Isso não é uma referência completa e sua versão CLI pode mostrar parâmetros ou comandos ligeiramente diferentes. Para obter uma visão geral dos recursos do Azure e grupos de recursos, consulte [Visão geral do Gerenciador de recursos do Azure](../azure-resource-manager/resource-group-overview.md).  

Para começar a, primeiro [instalar o CLI do Azure](../xplat-cli-install.md) e [se conectar à sua assinatura do Azure](../xplat-cli-connect.md) usando um trabalho ou conta de escola ou uma identidade de conta da Microsoft.

Sintaxe de comando atual e as opções na linha de comando no modo do Gerenciador de recursos, digite `azure help` ou, para exibir a Ajuda para um comando específico, `azure help [command]`. Também encontre exemplos CLI na documentação para criar e gerenciar serviços Azure específicos.

Parâmetros opcionais são mostrados entre colchetes (por exemplo, `[parameter]`). Todos os outros parâmetros são necessários.

Além de parâmetros opcionais do comando específico documentados aqui, há três parâmetros opcionais que podem ser usados para exibir a saída detalhada como opções de solicitação e códigos de status. O `-v` parâmetro fornece a saída detalhada e o `-vv` parâmetro fornece ainda mais detalhadas saída detalhada. O `--json` opção apresenta o resultado no formato json bruto.

## <a name="setting-the-resource-manager-mode"></a>Configuração do modo do Gerenciador de recursos

Use o seguinte comando para ativar os comandos de modo do Gerenciador de recursos do Azure CLI.

    azure config mode arm

>[AZURE.NOTE] Modo de Gerenciador de recursos do Azure e gerenciamento de serviço do Azure a CLI são mutuamente exclusivos. Isto é, criados em um modo de recursos não podem ser gerenciados do outro modo.

## <a name="azure-hdinsight-commands-to-manage-your-hdinsight-clusters"></a>Azure hdinsight: comandos para gerenciar seus clusters HDInsight

    hdinsight config create [options] <configFilePath> <overwrite>
    hdinsight config add-config-values [options] <configFilePath>
    hdinsight config add-script-action [options] <configFilePath>

Exemplo: Crie um arquivo de configuração que contém uma ação de script para executar ao criar um cluster.

    hdinsight config create "C:\myFiles\configFile.config"
    hdinsight config add-script-action --configFilePath "C:\myFiles\configFile.config" --nodeType HeadNode --uri <scriptActionURI> --name myScriptAction --parameters "-param value"

## <a name="azure-vm-commands-to-manage-your-azure-virtual-machines"></a>máquina virtual Azure: comandos para gerenciar suas máquinas virtuais do Azure

**Criar uma máquina virtual**

    vm create [options] <resource-group> <name> <location> <os-type>

**Criar uma máquina virtual com recursos padrão**

    vm quick-create [options] <resource-group> <name> <location> <os-type> <image-urn> <admin-username> <admin-password
    
>[AZURE.TIP]Começando com CLI versão 0,10, você pode fornecer um alias curto como "UbuntuLTS" ou "Win2012R2Datacenter" como o `image-urn` para algumas imagens do Marketplace populares. Executar `azure help vm quick-create` para as opções. Além disso, começando com a versão 0.10, `azure vm quick-create` usa o armazenamento de premium por padrão, se ele estiver disponível na região selecionado.

**Listar as máquinas virtuais dentro de uma conta**

    vm list [options]

**Obtenha uma máquina virtual dentro de um grupo de recursos**

    vm show [options] <resource-group> <name>

**Excluir uma máquina virtual dentro de um grupo de recursos**

    vm delete [options] <resource-group> <name>

**Desligar uma máquina virtual dentro de um grupo de recursos**

    vm stop [options] <resource-group> <name>

**Reiniciar uma máquina virtual dentro de um grupo de recursos**

    vm restart [options] <resource-group> <name>

**Iniciar uma máquina virtual dentro de um grupo de recursos**

    vm start [options] <resource-group> <name>

**Desligar uma máquina virtual dentro de um grupo de recursos e os recursos de computação de versões**

    vm deallocate [options] <resource-group> <name>

**Lista tamanhos de máquina virtual disponível**

    vm sizes [options]

**Capturar a máquina virtual como imagem de sistema operacional ou imagem de máquina virtual**

    vm capture [options] <resource-group> <name> <vhd-name-prefix>

**Definir o estado da máquina virtual para Generalized**

    vm generalize [options] <resource-group> <name>

**Obter o modo de exibição de instância da máquina virtual**

**Habilitar redefinir as configurações de acesso remoto a área de trabalho ou SSH em uma máquina Virtual e redefinir a senha da conta que tem o administrador ou sudo autoridade**
